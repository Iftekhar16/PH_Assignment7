What are the key differences between map, filter, forEach and find?

Answer:
1. The purpose of map is to take in an array and pass it through a function which manipulates the array and then returns a new array. The new array has the exact length of the previous array.
2. filter takes in an array and then passes it through a function as well. But this filters out tthe elemnts that are not "true", meaning it only returns the elemnets as a new array that meet the logic defined in the function.
3. forEach iterates for each and every element of the array and executes commands inside the function. It does not return a new array.
4. find is used to find the first element of the array that meet a certain criteria defined as a parameter. It returns only that element.

What are the key differences between local storage, session storage and cookies?

Answer:
1. Local storage is persistent, meaning the data will be stored within the browser and does not go away if the borwser is closed. The storage limit here is typically 5-10MB. These storages are usually used to store user's preferences and cached data.
2. Session storage is not persistent but rather session based, meaning the data will be removed if the tab or browser is closed. This also has the same storage limit as local storage which is around 5-10MB. These are usually used to store temporary data or page specific short term information.
3. Cookies can be both persistent and session based. These have a very short storage, typically around 4KB per cookie. These are used to store some small tokens or information that is needed for website functionality.

Answer: Event loop is a conccept of asynchronus programming. It continuously keeps checking the call stack and executes pending tasks in a non-blocking manner which then allows the website/web page to be responsive and not laggy.
